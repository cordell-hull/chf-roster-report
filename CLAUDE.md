# Teacher Roster

Static web app (no build step) for host schools to manage their J-1 teacher roster and generate PDF reports. Runs entirely in the browser using ES modules and pdf-lib. Supports multiple teachers and their accompanying relatives (for J-2 visas).

## Local Development

```bash
python3 -m http.server 8083
# Open http://localhost:8083
```

Must be served over HTTP — `file://` won't work due to ES module imports.

## Versioning

Version is defined in `config/version.js` as the single source of truth. It is displayed in the page footer and included in debug reports.

**When releasing changes**, update both:
1. `APP_VERSION` in `config/version.js`
2. The `?v=` query strings on `<link>` and `<script>` tags in `index.html`

## Key Files

- `index.html` — Main page, 5-step wizard (School, Teachers, Relatives, Sign, Review)
- `main.js` — Wizard logic, teacher/relative table management, import from recruitment, persistent storage
- `lib/pdf.js` — PDF generation with pdf-lib (landscape layout, teacher table + relatives table + signature)
- `config/org.js` — Organization branding, PDF styling constants
- `config/version.js` — App version constant
- `styles.css` — All styles (shared base with recruitment report)

## Shared localStorage

This app shares data with other CHF forms via localStorage:
- `chf-school-info` — School name and contact info (read/write, shared across all CHF forms)
- `chf-recruitment-data` — Recruitment report data (read-only, used to import teachers)
- `chf-roster-data` — This app's data (teachers, relatives, signature)

All apps are deployed under `cordell-hull.github.io/*` so they share the same origin.

## Data Persistence

All roster data is saved to `localStorage` under key `chf-roster-data` — on every step transition, teacher/relative save, and every 30 seconds via auto-save. On page load, if saved data exists, a landing screen offers "Continue" or "Start New Roster". Data persists across sessions and PDF downloads.

## Import Features

- **Import from Recruitment Report**: Reads teachers from `chf-recruitment-data` in localStorage. Pre-fills name, email, and citizenship. Only imports teachers not already in the roster (matched by email).
- **Import from PDF**: Reads embedded JSON metadata from PDFs generated by this app.

## PDF Generation Notes

- Landscape layout (792x612 points) to fit wide teacher tables.
- Uses smaller fonts (6-8pt) for table cells.
- Teacher table with 15 columns, auto-truncates long values.
- Relatives table follows with J-2 visa note.
- Headers re-drawn on page breaks.
- Report data embedded in PDF Subject metadata for re-import.
